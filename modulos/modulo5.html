Lección 20 30 min

I/O Moderno (NIO.2)

Manejo eficiente de archivos y directorios con `java.nio.file`. Olvida `java.io.File`.

Objetivos de Élite

Superar `java.io.File` con `Path` y `Files`.
Streams perezosos vs Lectura voraz.
Gestión de recursos `try-with-resources`.
Manipulación agnóstica al S.O.

1. Teoría Suprema: Hablando con el Disco

El disco duro es millones de veces más lento que la CPU. NIO.2 (New I/O 2) es la API moderna para interactuar con el sistema de archivos de forma eficiente y segura.

¡Peligro de Memoria!

Nunca uses Files.readAllLines() para archivos cuyo tamaño desconoces. Carga todo en RAM y provoca OutOfMemoryError. Usa Files.lines(), que devuelve un Stream procesado línea a línea.

2. Laboratorio de Código

ArchivosModernos.java 
import java.nio.file.*;
import java.util.stream.Stream;

// 1. Path define la ruta (funciona en Win/Linux/Mac)
Path ruta = Path.of("datos", "logs", "server.log");

// 2. Escritura rápida
Files.createDirectories(ruta.getParent());
Files.writeString(ruta, "INFO: Inicio\nERROR: Fallo Crítico");

// 3. Lectura Eficiente (Lazy Stream)
// try-with-resources cierra el archivo automáticamente
try (Stream lineas = Files.lines(ruta)) {
    lineas.filter(l -> l.startsWith("ERROR"))
          .forEach(System.out::println);
}

3. Entrenamiento Progresivo

NIVEL BÁSICO

Verificación de Existencia

Usa `Files.exists(path)` para verificar un archivo. Si no existe, créalo con `Files.createFile`.

NIVEL EXPERTO

Crawler de Archivos

Usa `Files.walk(path)` para encontrar recursivamente todos los archivos `.java` en una carpeta.

try (Stream paths = Files.walk(Path.of("src"))) {
    paths.filter(Files::isRegularFile)
         .filter(p -> p.toString().endsWith(".java"))
         .forEach(System.out::println);
}
Lección 21

Concurrencia Básica

Hilos, memoria compartida y el caos de las Race Conditions.

1. Teoría Suprema: El Caos de la Cocina

Si dos cocineros (Hilos) intentan cortar en la misma tabla (Objeto) sin coordinarse, los datos se corrompen. Esto es una Race Condition.

La Solución: Synchronized

La palabra clave synchronized actúa como un candado en la puerta del método. Solo un hilo puede entrar a la vez.

public class Contador {
    private int cuenta = 0;

    // El candado evita que dos hilos escriban a la vez
    public synchronized void incrementar() {
        cuenta++;
    }
}
Lección 22

Virtual Threads (Loom)

La revolución de Java 21: Millones de hilos ligeros.

1. Teoría Suprema: Project Loom

Antes, 1 Hilo Java = 1 Hilo del Sistema Operativo (Pesado). Loom introduce hilos gestionados por la JVM que son extremadamente ligeros.

Modelo "Uber"

Los hilos virtuales son tareas que comparten un número limitado de hilos del sistema (el "coche"). Cuando una tarea espera (duerme), se baja del coche y deja que otra lo use.

// Ejecutor moderno de Java 21
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    // Podemos lanzar 100,000 tareas sin colapsar la RAM
    for (int i = 0; i < 100_000; i++) {
        executor.submit(() -> {
            System.out.println("Hilo virtual corriendo");
        });
    }
} // Espera automática a que terminen todas
Lección 23

Testing con JUnit 5

Asegura la calidad de tu código y duerme tranquilo.

1. Teoría Suprema: Ciclo de Vida del Test

Un test unitario aísla la unidad más pequeña de código.

class CalculadoraTest {
    private Calculadora c;

    @BeforeEach // Se ejecuta antes de CADA test
    void setup() { c = new Calculadora(); }

    @Test
    void sumarDebeFuncionar() {
        assertEquals(5, c.sumar(2, 3));
    }

    @Test
    void divisionPorCeroLanzaExcepcion() {
        assertThrows(ArithmeticException.class, () -> c.dividir(1, 0));
    }
}
