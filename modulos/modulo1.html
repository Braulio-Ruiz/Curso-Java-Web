La Máquina Virtual (JVM) y el Ecosistema

Objetivos de Élite

Entender la JVM como S.O. virtual.
Compilación JIT y Bytecode.
Arquitectura Stack vs Heap.
Diferencias JDK vs JRE.

1. Teoría Suprema: Más allá del "Hello World"

Muchos cursos te enseñan a conducir el coche (sintaxis), pero aquí vamos a abrir el capó para entender el motor. Java es famoso por su lema "Write Once, Run Anywhere" (WORA). ¿Cómo es esto posible si cada procesador habla un idioma distinto?

1.1 El Esperanto del Software: Bytecode

Cuando compilas (`javac`), no creas un ejecutable (`.exe`) para Windows. Creas un archivo `.class` que contiene Bytecode.

Analogía del Traductor

Tú eres el autor (Programador) y escribes un libro en un idioma perfecto (Java). La JVM es un traductor simultáneo experto que vive en la computadora del usuario. Lee tu libro (Bytecode) y lo traduce en tiempo real al dialecto local de la CPU.

1.2 Arquitectura de Memoria

The Stack (La Pila)

Mesa de trabajo rápida y ordenada. Tareas inmediatas y primitivos. Gestión LIFO.

The Heap (El Montículo)

Gran almacén desordenado. Aquí viven los OBJETOS. Gestionado por el Garbage Collector.

2. Laboratorio de Código

ArquitecturaMemoria.java 
public class ArquitecturaMemoria {
    // 'main' entra al Stack Frame principal
    public static void main(String[] args) {
        int id = 10; // Primitivo: Vive directamente en el Stack
        
        // 'usuario' es una Referencia en el Stack.
        // El objeto real {nombre: "Ana"} se crea en el Heap.
        Usuario usuario = new Usuario("Ana"); 
        
        crearReporte(usuario); // Nuevo frame apilado en el Stack
    } // Stack se vacía, GC limpia el Heap.

    public static void crearReporte(Usuario u) {
        // ... lógica
    }
}

Dominando IntelliJ IDEA

Objetivos de Élite

1. Teoría Suprema: Tu IDE es tu Exosqueleto

Un programador senior no escribe código más rápido tecleando; lo hace porque el IDE escribe el 40% por él. IntelliJ IDEA no es un editor de texto, es un motor de análisis estático.

1.1 Indexado: El Cerebro Artificial

Cuando abres un proyecto y ves "Indexing...", IntelliJ está creando un mapa mental de todo tu código. Esto permite:

  • Refactorización Segura: Renombrar una clase actualiza todas sus referencias.
  • Análisis de Código Muerto: Te avisa si un método no se usa.

2. Laboratorio de Productividad

public class Productividad {
    // Escribe 'psvm' + TAB
    public static void main(String[] args) {
        String saludo = "Hola Senior";
        
        // Escribe 'sout' + TAB
        System.out.println(saludo);

1. Teoría Suprema: Bits y Precisión

Elegir el tipo de dato incorrecto puede costar millones (errores de redondeo) o tumbar servidores (memoria).

Tipo Tamaño Uso Ideal
int 32 bits Contadores, índices.
long 64 bits IDs de BD, timestamps.
double 64 bits Científico (NO dinero). 
import java.math.BigDecimal;

public class Precision {
    public static void main(String[] args) {
        // EL CRIMEN FINANCIERO
        double precio = 0.1;
        System.out.println(precio + 0.2); // 0.30000000000000004
        
        // LA SOLUCIÓN PROFESIONAL
        BigDecimal exacto = new BigDecimal("0.1");
        BigDecimal suma = exacto.add(new BigDecimal("0.2"));
        System.out.println(suma); // 0.3
    }
}
Lección 04

Control de Flujo

1. Teoría Suprema: La Carga Cognitiva

Cada `if` anidado aumenta la carga cognitiva. Un código con forma de flecha (Arrow Code) es difícil de mantener.

Patrón: Guard Clauses

Mal: `if (user != null) { if (active) { ... } }`
Bien: `if (user == null) return;` `if (!active) return;`

Lección 05

Métodos y Clean Code

1. Teoría Suprema: The Stack Frame

Cuando llamas a un método, Java crea un "Stack Frame" en la Pila con sus variables locales. Cuando termina, se destruye.

¡Java SIEMPRE pasa por valor!

Se pasa el valor de la referencia (una copia de la dirección), no el objeto en sí. Puedes modificar el interior del objeto, pero no cambiar a qué objeto apunta la variable original.

// Un método debe hacer UNA sola cosa.
public static double calcularSalario(double bruto) {
    double impuesto = calcularIVA(bruto); // Delegar
    return bruto - impuesto;
}
