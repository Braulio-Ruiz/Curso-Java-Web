Arrays y Strings

La física de la memoria: por qué el acceso a Arrays es instantáneo y cómo evitar matar el rendimiento con Strings.

Objetivos de Élite

Física de Arrays: Memoria contigua y O(1).
String Pool y Optimización de Memoria.
StringBuilder vs Concatenación (+).
Java Moderno: Text Blocks.

1. Teoría Suprema: La Memoria y el Texto

Los Arrays son la estructura de datos más básica y eficiente. Son bloques de memoria contigua. Si sabes dónde empieza y el tamaño del dato, saltar a la posición 1000 es instantáneo.

1.1 Inmutabilidad de Strings

El Pecado Capital del Rendimiento

Hacer texto += "a" dentro de un bucle de 10,000 iteraciones crea 10,000 objetos intermedios que el Garbage Collector debe limpiar. Esto congela aplicaciones.

2. Laboratorio de Código

Optimizacion.java 
public class Optimizacion {
    public static void main(String[] args) {
        // String Pool: 'a' y 'b' apuntan al MISMO objeto.
        String a = "Java";
        String b = "Java"; 
        System.out.println(a == b); // true

        // ELITE: StringBuilder usa un array interno redimensionable.
        StringBuilder rapido = new StringBuilder();
        for (int i = 0; i < 1000; i++) {
            rapido.append(i); // Modifica el mismo objeto, cero basura.
        }
        
        // Java 15+: Text Blocks (Ideal para JSON/SQL)
        String json = """
              {
                  "id": 1,
                  "status": "ACTIVE"
              }
              """;
    }
}

Algoritmo de Rotación (O(1) Espacio)

Rota un array `k` veces a la derecha sin crear arrays auxiliares. Pista: Invertir partes del array.

public void rotar(int[] nums, int k) {
    k %= nums.length;
    invertir(nums, 0, nums.length - 1); // Todo
    invertir(nums, 0, k - 1);           // Parte 1
    invertir(nums, k, nums.length - 1); // Parte 2
}
// Método auxiliar 'invertir' usa dos punteros (start/end)
Lección 12

Manejo de Excepciones

Errores recuperables vs Bugs. Jerarquía y gestión de recursos.

1. Teoría Suprema: No es solo un "Error"

Crear una excepción es costoso para la JVM (Stack Trace). Úsalas para casos excepcionales, no para lógica de negocio.

[Image of Java Exception Class Hierarchy]

Checked (Exception)

El compilador OBLIGA a manejarlas. Para errores externos recuperables (Red, Archivos).

Unchecked (Runtime)

Errores de programación (Bugs). No se capturan, se arreglan (NullPointer).

public void leerDatos() throws IOException {
    // Try-with-resources: Cierra el archivo automáticamente
    try (BufferedReader br = new BufferedReader(new FileReader("data.txt"))) {
        System.out.println(br.readLine());
    } 
    // No hace falta finally para cerrar
}
Lección 13

Collections: List y Set

1. Teoría Suprema: Ingeniería de Datos

Elegir la colección incorrecta puede degradar el rendimiento masivamente.

La Mentira de LinkedList

Aunque insertar es O(1), llegar a la posición es O(N). Además, los nodos dispersos causan "Cache Misses" en la CPU. ArrayList es casi siempre superior.

La Magia del Hash (Set)

HashSet usa el hashCode() para saltar directamente a la posición de memoria.

// LIST: Ordenada, duplicados OK
List lista = new ArrayList<>();

// SET: Sin orden, ÚNICOS (Rapidísimo para búsquedas)
Set unicos = new HashSet<>();
unicos.add(1);
unicos.add(1); // Ignorado
System.out.println(unicos.size()); // 1
Lección 14

Map y Queue

Motor Clave-Valor

HashMap es la estructura más usada. Es vital entender cómo maneja las colisiones (cuando dos claves caen en el mismo bucket).

Map inventario = new HashMap<>();
inventario.put("Manzana", 10);

// Iteración eficiente
for (Map.Entry entry : inventario.entrySet()) {
    System.out.println(entry.getKey() + ": " + entry.getValue());
}

// PriorityQueue: El menor sale primero (Ideal para tareas)
Queue cola = new PriorityQueue<>();
cola.add(50); cola.add(10);
System.out.println(cola.poll()); // 10
Lección 15

Genéricos

Seguridad de tipos en compilación y flexibilidad de API.

1. Teoría Suprema: Type Erasure

Para mantener compatibilidad, Java BORRA la información de tipos al compilar. List<String> se convierte en List (raw) en el bytecode.

Principio PECS

"Producer Extends, Consumer Super". Usa ? extends T si solo lees de la lista. Usa ? super T si escribes en ella.

public class Caja {
    private T contenido;
    public void guardar(T c) { this.contenido = c; }
    public T sacar() { return contenido; }
}

// Comodines (Wildcards)
public double sumar(List nums) {
    double suma = 0;
    for (Number n : nums) suma += n.doubleValue();
    return suma;
}
