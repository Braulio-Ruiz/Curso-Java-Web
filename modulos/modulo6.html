SQL y JDBC

La base de todo backend profesional: conectar, consultar y transaccionar con bases de datos de forma segura.

Objetivos de Élite

Entender Drivers, Connections y Statements.
Prevenir Inyección SQL (PreparedStatement).
Gestión de Transacciones (ACID).
Patrón DAO (Data Access Object).

1. Teoría Suprema: El Puente de Datos

JDBC es la API de bajo nivel sobre la que se construyen todos los frameworks modernos. Java habla métodos, y el Driver traduce esos métodos al protocolo nativo de la Base de Datos (MySQL, PostgreSQL, etc).

[Image of SQL Database connection diagram]

Peligro Mortal: Inyección SQL

Jamás concatcnes Strings en una consulta SQL. Si haces "SELECT * FROM users WHERE pass = '" + pass + "'", un atacante puede entrar sin contraseña.

Solución: Usa siempre PreparedStatement.

2. Laboratorio de Código

UsuarioDAO.java 
public class UsuarioDAO {
    private static final String SQL = "SELECT id, nombre FROM usuarios WHERE rol = ?";

    public List buscarPorRol(String rol) {
        List usuarios = new ArrayList<>();

        // try-with-resources cierra Connection y Statement automáticamente
        try (Connection conn = DriverManager.getConnection(URL, USER, PASS);
             PreparedStatement stmt = conn.prepareStatement(SQL)) {

            // 1. Asignación Segura (Previene Inyección)
            stmt.setString(1, rol);

            // 2. Ejecución
            try (ResultSet rs = stmt.executeQuery()) {
                // 3. Mapeo Relacional -> Objetos
                while (rs.next()) {
                    usuarios.add(new Usuario(
                        rs.getLong("id"),
                        rs.getString("nombre")
                    ));
                }
            }
        } catch (SQLException e) {
            throw new RuntimeException("Error en BD", e);
        }
        return usuarios;
    }
}

NIVEL EXPERTO

Transacción Bancaria Atómica

Simula una transferencia (Resta a A, Suma a B). Usa `setAutoCommit(false)`, `commit()` y `rollback()` para asegurar que el dinero no se pierda si hay error.

try (Connection conn = DriverManager.getConnection(URL)) {
    conn.setAutoCommit(false); // Inicio Transacción
    try {
        // ... update restar ...
        // ... update sumar ...
        conn.commit(); // Confirmar
    } catch (Exception e) {
        conn.rollback(); // Deshacer todo
    }
}

Lección 26

Intro a JPA y ORM

Automatiza el acceso a datos mapeando objetos a tablas.

1. Teoría Suprema: La Magia del ORM

JPA (Java Persistence API) automatiza el trabajo de JDBC. Un concepto crítico es el Persistence Context: una caché donde los objetos están "vigilados" (Managed).

[Image of ORM mapping diagram]

Eager vs Lazy Loading

Eager: Carga todo (Usuario + 1000 pedidos). Lento.
Lazy: Carga solo el Usuario. Los pedidos se traen solo si los pides. (Recomendado). 

@Entity
public class Empleado {
    @Id @GeneratedValue
    private Long id;
    private String nombre;
}

// Uso
em.getTransaction().begin();
Empleado e = new Empleado("Ana");
em.persist(e); // Managed
e.setNombre("Ana María"); // JPA detecta el cambio y hará UPDATE solo
em.getTransaction().commit();
Lección 27

JSON (Jackson)

El lenguaje universal para comunicar tu backend con el mundo.

1. Teoría Suprema: La Lingua Franca

Tus objetos viven en RAM. Para enviarlos a un Frontend, deben viajar como texto JSON. Jackson es la librería estándar para esto.

Anti-Patrón: Exponer Entidades

Nunca devuelvas una `@Entity` JPA en un controlador REST. Podrías exponer contraseñas o causar bucles infinitos. Usa DTOs (Records). 

// DTO Inmutable
public record UsuarioDTO(String nombre, String email, @JsonIgnore String password) {}

// Serialización
ObjectMapper mapper = new ObjectMapper();
UsuarioDTO user = new UsuarioDTO("Carlos", "c@test.com", "123");
String json = mapper.writeValueAsString(user);
// Resultado: {"nombre":"Carlos","email":"c@test.com"} (Sin password)
